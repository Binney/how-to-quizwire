More complex Javascript commands

We'll now cover three features of programming languages which are very useful: if, for, and while.

Create a new web page for these commands.

if

You don't always want your code to do the same thing. What if you want something different to happen based on what the user's name is, or the day of the week? The answer is, you'd use an if statement. They look like this:

if (1 + 1 === 2) { console.log("Maths still works!"); }

Let's look a bit more closely at the bit in the round brackets, ( ). This is called the condition. Think about what possible values that could be. Either 1+1 equals 2, or it doesn't. There's no "maybe" about it; there are only 2 possible options. In computing, instead of saying "yes" and "no", we say true and false. There are lots of conditions we could put in those brackets. For example:

var turboModeEnabled = false; if (turboModeEnabled) { console.log("Turbo mode is enabled!"); }
var myName = "Sarah"; if (myName === "Mickey Mouse") { console.log("Squeak"); }

There's something rather weird about that last bit. Count the number of equals (=) signs on each line. You'll see there's only one equals sign on the first line, but three on the second. What's going on? The first line is assigning a value of "Sarah" to the myName variable, so we use one equals sign. The second line is checking whether two variables are the same, so we use three.

If there's one equals sign, we're assigning a value. If there are three equals signs, we're checking if two things are the same or not.
